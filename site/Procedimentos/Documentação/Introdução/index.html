    Introdução

    Objetivo

    • Facilitar a troca de informação entre os setores;
    • Padronização;
    • Correção de erros;
    • Aprendizado contínuo.

    Pré requisitos para elaboração da documentação

    • Conhecimento básico em markdown
    • Instalação do python
    • Instalação do MKDocs

    Markdown

    Os arquivos da documentação são escritos em markdown.
    Markdown é uma linguagem de marcação que utiliza simbolos para formação, os principais simbolos podem ser consultados no cheat sheet.

    MKDocs

    Essa documentação é escrita com markdown e gerada pelo MKDocs, uma ferramenta de feração de sites estáticos.

    O MKDocs transforma os arquivos escritos em Markdown em páginas web.

    Para melhor apresentação da documentação o tema escolhido foi o Material.

    Pré requisitos

    Para utilização do MKDocs é necessário ter o python e o gerenciador de pacotes pip instalados.
    O python pode ser baixado na página de downloads.

    Instalação do MKDocs

    No terminal / cmd digite:

        pip install mkdocs

    Para instruções mais detalhadas e solução de dúvidas, vá ao site de instalação do MKDocs

    Utilização do MKDocs

    Criação da documentação

    Para rodar o https://www.mkdocs.org/, no terminal / cmd rode o comando:

        mkdocs new nome_da_pasta_de_documentação

    Para esse projeto foi utilizada a pasta padrão do https://www.mkdocs.org/.

        mkdocs new .

    Após rodar o comando, o https://www.mkdocs.org/ irá criar uma pasta .docs, onde serão armazenadas todos os arquivos da documentação.

    Vizualização da documentação

    Para vizualizar a documentação criada, rode o comando:

        mkdocs serve

    O comando irá rodar um servidor local (na máquina do usuário) onde a documentação pode ser vizualizada.

    Gerar a documentação

    Para que a documentação seja acessível para todos os usuários ela deve ser independete, ou seja, os usuários que irão vizualizar a documentação não devem se preocupar com os pré requisitos.
    Para isso, após o término da documentação rode o comando:

        mkdocs build

    O comando transformará todos os arquivos https://www.markdownguide.org/ em arquivos html que podem ser acessados por todos os navegadores, esses arquivos serão criados na pasta site.

    Material

    O material é um tema aplicado ao MKDocs para alterar o layout das páginas e adicionar algumas funcionalidades. O material é o tema mais utilizado nas documentações atuais.

    Instalação do Material

    No terminal / cmd digite:

        pip install mkdocs-material

    Para mais detalhes sobre o tema acesse a página do [material para MKDocs]https://squidfunk.github.io/mkdocs-material/publishing-your-site/ no [Github]https://github.com/

    Plugins

    MKDocs macros

    O MKDcos macros permite a criação de variáveis no arquivo mkdocs.yml do MKDocs para facilitar futuras manutenções.

    Instalação

    No terminal / cmd digite:

        pip install mkdocs-macros-plugin

    Utilização

    extra:
    nome_grupo:
        nome_variavel: valor_variavel

    extra:
    nomes:
        perfil_viga: perfil U

    Referência:

        O {{ no such element: dict object['perfil_viga'] }}

    Resultado:

        O perfil U...

    pymdown-extension

    O pymdown extension é utilizado para disponibilizar os recursos de https://www.markdownguide.org/ expandido, como texto realçado, texto tachado, [ ] ToDo list, entre outros recursos.

    Instalação

    No terminal / cmd digite:

        pip install mdown-extension-plugin

    Utilização

    Para adicionar os recursos desejados, no arquivo .yml, no grupo markdown_extensions, adicione os recursos desejados.
    Para consulta dos recursos e mais informações acesse a documentação do PyMDown Exntensions

    Disponibilar a documentação online

    Todos os arquivos referentes a documentação estão disponíveis no repositório documentacao_gimi do github.

    Após qualquer atualização ou ao fim do dia de trabalho é recomendado o upload da documentação no repositório do github.

    O repositório do projeto está vinculado com o netlify, um site de hospedagem de sites estáticos, onde a documentação pode ser acessada por qualquer dispositivo.

    Atualização da documentação

    Ao terminar a revisão da documentação é necessário atualiza-la no repositório do github, a atualização pode ser feita por dois métodos.

    1. Método pela linha de comando

    1.1 Abra o terminal / cmd

    Altere o diretório para a pasta raiz da documentação utilizando o comando cd

        cd "pasta_da_documentação"

    Como exemplo imagine que seus arquivos de documentação estão na pasta Documentação GIMI na sua área de trabalho.

        cd "C:\Users\user1\Área de Trabalho\Documentação GIMI"

    1.2 Adicione os arquivos alterados
    Os arquivos são adicionados ao commit com o comando git add e o nomes dos arquivos que serão adicionados com as extensões, para mais de uma arquivo os nomes devem ser separados por um espaço.

    No terminal / cmd digite o comando

        git add arquivo1.md arquivo2.md

    1.3. Verificação do Status dos arquivos adicionados
    Para verificar se os arquivos foram adicionados ao commit use o comando status.

    No terminal / cmd digite o comando

        git status

    1.4 Commit
    Para fazer o upload dos arquivos no repositório do github, use o comando commit
    Para os commits, dese-se seguir o padão mostrado abaixo. 

        git commit -m "mesagem_do_cimmit"

    A mennsagem do commit deve seguir o padrão mostrado no tópico sobre mensagem de commit.

    2. Método pelo VSCode

    2.1 Adicionando os arquivos alterados
    No VSCode, no menu lateral clique na ícone de controle de versão (Source Control).

    Os arquivos alterados serão mostrados no campo change.
    Para adicionar os arquivos individualmente no commit clique no "+" a direita do nome do arquivo.

    Para adionar todos os arquivos do campo change clique no "+" ao lado do nome do campo.

    2.2 Commit

    Para fazer o upload dos arquivos no repositório do https://github.com/, escreva a mensagem na caixa de texto (Message) com o padrão mostrado no tópico sobre mensagem de commit

    Mensagem de Commit

    Os emojis e categorias padronizados podem ser encontrados na página de padrões de commit no github.
    A mensagem padrão segue o seguinte layout:

        :emoji: categoria: "mensagem de commit"
